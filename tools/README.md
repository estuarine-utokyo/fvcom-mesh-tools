# tools/

Maintenance scripts for `fvcom-mesh-tools`. None of these are part of
the installed Python package; they are operational utilities run from
the repo root.

## OceanMesh2D data migration

The Tokyo Bay datasets that originally lived under
`$HOME/Github/OceanMesh2D/{datasets,Tokyo_Bay/data,tb_mesh}` are
relocated to the group-shared `$DATA_DIR` so that:

- both `oceanmesh-tools` and `fvcom-mesh-tools` can reference one
  canonical copy via symlinks,
- OceanMesh2D itself remains usable as a comparison baseline (its
  paths continue to resolve through symlinks),
- when OceanMesh2D is eventually retired, the data are unaffected.

### Resulting layout (under `$DATA_DIR`)

```
$DATA_DIR/
├── bathymetry/
│   ├── GEBCO/         GEBCO_2024.nc
│   ├── SRTM15plus/    SRTM15+.nc
│   ├── CUDEM/         CUDEM_*.nc
│   └── tokyo_bay/     dem_*.nc, depth_*.nc       (was OceanMesh2D/datasets/TokyoBay/dem)
├── coastline/
│   ├── GSHHS/         {c,f,h,i,l}/                (global shoreline)
│   └── tokyo_bay/
│       ├── MLIT_C23/   C23-06_TOKYOBAY{,_INNER,_OUTER}.*
│       ├── Futtsu/     Futtsu_coastline.*, coastline_2.*
│       ├── SMS/        (as-is from OceanMesh2D)
│       ├── Wang/       (was Wang_coastline)
│       └── misc/       tokyo.nc, tokyo_0001.nc
└── mesh/
    ├── reference/     OceanMesh2D-generated, read-only baseline
    │   └── tokyo_bay/  tb_futtsu*.14 / *.mat / source/genMesh_*.m
    └── fvcom/         meshes generated by this project
        └── tokyo_bay/
```

The `fvcom-mesh-tools` repo accesses these via three top-level symlinks
under `data/` (gitignored):

```
data/bathymetry -> $DATA_DIR/bathymetry
data/coastline  -> $DATA_DIR/coastline
data/mesh       -> $DATA_DIR/mesh
```

### Migration workflow

The migration is split into three explicit phases. **Do not skip the
verification step** — the symlink replacement is destructive.

#### Phase 1: copy + verify (batch, non-destructive)

```bash
pjsub tools/migrate_oceanmesh2d_data.pjsub
```

The job:
1. Creates the destination directories under `$DATA_DIR`.
2. `rsync`s every dataset from OceanMesh2D into its target location.
3. Verifies the copies via `diff -rq` on directories and `md5sum` on
   the large NetCDF / fort.14 / mat files.

Inspect the job log; every entry should read `[verify] OK`. If any
`FAIL` appears, fix the cause and re-run (`rsync` is idempotent; the
job is safe to re-submit).

Resource footprint: 1 vnode-core, 1 hour wall (typically completes in
a few minutes once the queue is dequeued; the bulk of the time is the
GEBCO + SRTM transfer ≈ 10 GB).

#### Phase 2: replace originals with symlinks (destructive, login node)

After Phase 1 verification passes:

```bash
# Dry-run first (the default) — prints what would change.
bash tools/replace_oceanmesh2d_with_symlinks.sh

# When the dry-run looks correct, apply.
bash tools/replace_oceanmesh2d_with_symlinks.sh --apply
```

This `rm -rf`s each original path under
`$HOME/Github/OceanMesh2D/{datasets,Tokyo_Bay/data,tb_mesh}` and
replaces it with a symlink that points into `$DATA_DIR`. After this
runs, the OceanMesh2D MATLAB scripts are unmodified and continue to
read the same paths transparently.

#### Phase 3: wire up `fvcom-mesh-tools/data/` (login node)

```bash
cd "${HOME}/Github/fvcom-mesh-tools"
mkdir -p data
ln -s "${DATA_DIR}/bathymetry"  data/bathymetry
ln -s "${DATA_DIR}/coastline"   data/coastline
ln -s "${DATA_DIR}/mesh"        data/mesh
```

`data/` is in `.gitignore`, and the symlinks themselves are not
tracked.

### Per-source PROVENANCE.md (recommended, manual)

After the data lands in `$DATA_DIR`, drop a small `PROVENANCE.md` into
each source directory recording origin, acquisition date, and license.
Example for `$DATA_DIR/bathymetry/GEBCO/PROVENANCE.md`:

```markdown
# GEBCO 2024
Source:    https://www.gebco.net/data_and_products/gridded_bathymetry_data/
Acquired:  YYYY-MM-DD by <name>
License:   Free for non-commercial use; commercial use requires permission
           from the GEBCO Compilation Group. See license at the source URL.
File:      GEBCO_2024.nc  (15 arc-second global bathymetry/topography)
Used by:   OceanMesh2D (legacy), fvcom-mesh-tools (active)
```
